摘   要
（小二、黑体、居中，样式LW居中标题：ALT+C）

　　随着移动互联网的普及和优生优育观念的深入人心，准妈妈群体对于专业孕期知识获取、同理心情感交流以及经验分享的需求日益增长。现有的通用社交平台难以通过垂直化的服务满足该群体对隐私保护、即时互动和精准内容分发的需求。针对这一痛点，本文设计并实现了一个基于 Web 的“准妈妈孕期交流与分享平台”。
　　本系统采用 B/S 架构，前端基于 Next.js 框架和 React 库，利用 TypeScript 进行类型安全的开发，界面采用 Tailwind CSS 实现响应式布局；后端基于 Next.js API Routes 构建，使用 Prisma ORM 操作 MySQL 数据库；系统引入 Clerk 进行安全的身份认证，利用 Cloudinary 实现多媒体资源的云端存储。
　　系统核心功能包括用户身份管理、孕期动态图文发布、实时互动评论与点赞、宝妈社交关系链构建（关注/好友）、一对一私信即时通讯以及全文检索功能。通过引入乐观 UI 更新（Optimistic UI）和服务器端组件（RSC）技术，大幅提升了页面加载速度和交互响应体验。测试结果表明，该系统功能完善、运行稳定、操作便捷，能够有效为准妈妈群体提供一个温馨、便捷、互助的线上交流空间。

Abstract
（小二、黑体、居中，样式LW居中标题：ALT+C）

   With the popularity of mobile internet and the deepening concept of eugenics, the demand among expectant mothers for professional pregnancy knowledge, empathetic emotional exchange, and experience sharing is growing day by day. Existing general social networks struggle to meet the specific requirements of this group for privacy protection, instant interaction, and precise content distribution through vertical services. Addressing this pain point, this thesis designs and implements a Web-based "Communication and Sharing Platform for Expectant Mothers".
   The system adopts a B/S architecture. The front end is built on the Next.js framework and React library, utilizing TypeScript for type-safe development, and employs Tailwind CSS for responsive layout. The back end is constructed based on Next.js API Routes, using Prisma ORM to manipulate the MySQL database. The system integrates Clerk for secure identity authentication and utilizes Cloudinary for cloud storage of multimedia resources.
   Core functions of the system include user identity management, posting of pregnancy updates (text and images), real-time interactive comments and likes, construction of social relationship chains (following/friends), one-on-one private messaging, and full-text retrieval. By introducing Optimistic UI updates and React Server Components (RSC), the page loading speed and interactive response experience have been significantly improved. Test results show that the system is fully functional, stable in operation, and easy to use, effectively providing a warm, convenient, and supportive online communication space for expectant mothers.

1.1 研究背景与意义
（小三，宋体，加粗，样式LW标题2：ALT+2）
[标题后空一行]
　　孕期是女性生命中一段特殊且敏感的时期，生理和心理的剧烈变化使得准妈妈们迫切需要一个能够倾诉烦恼、分享喜悦、获取经验的平台。虽然市面上已有微信群、小红书等渠道，但微信群信息流杂乱且检索困难，通用社交平台则存在广告泛滥、隐私保护不足等问题。
　　设计一个垂直领域的“准妈妈孕期交流与分享平台”，不仅有助于缓解孕期焦虑，构建互大的互助社区，还能通过技术手段沉淀高质量的孕育经验，具有重要的社会意义和应用价值。

1.2 国内外研究现状
　　
　　目前，国外的垂直母婴社区如 BabyCenter 侧重于知识库构建，社交互动相对传统；国内的宝宝树、妈妈网等平台功能大而全，集成了大量电商导购属性，导致用户在纯粹交流体验上受到干扰。随着 Web 技术的发展，基于 Next.js 等现代框架构建轻量级、高性能、重交互的垂直社区成为技术发展的新趋势，能够更好地平衡用户体验与开发效率。
　　
1.3 主要研究内容
　　
　　本文主要研究内容包括：
1. 系统架构设计：搭建基于 Node.js 环境的 Next.js 全栈架构，实现前后端分离开发模式的统一管理。
2. 核心业务功能实现：开发动态发布流（Feed）、社交关系管理、实时消息系统等模块，适配孕期交流场景。
3. 技术难点攻关：解决服务端渲染（SSR）与客户端交互的数据同步问题，优化图片视频上传性能，实现基于 Prisma 的复杂关系查询。
4. 系统测试与部署：完成功能测试、兼容性测试，并进行系统部署。
第二章 相关技术综述
2.1 前端技术

　　Next.js (App Router)：介绍为什么选 SSR/RSC（有利于 SEO，首屏加载快，适合内容分享）。Next.js 是基于 React 的生产级框架，本系统采用其最新的 App Router 模式。它支持服务器端渲染（SSR）和静态站点生成（SSG），显著提升了首屏加载速度，有利于 SEO 优化，非常适合构建内容驱动型的社交平台。
　　React & TypeScript：组件化开发与类型安全。TypeScript 是 JavaScript 的超集，引入了静态类型系统。在本项目中，TypeScript 用于定义 API 接口返回结构、组件 Props 和数据库模型类型，在编译阶段即可发现潜在错误，极大提高了代码的可维护性和健壮性。
　　Tailwind CSS：原子化 CSS 的优势。Tailwind CSS 是一个功能类优先的 CSS 框架。系统无需编写传统的 .css 文件，而是通过 HTML 类名直接构建响应式界面，确保了整个平台 UI 风格的一致性与开发的高效性。

第三章 系统需求分析
　　
3.1 可行性分析
　　
1. 技术可行性：Next.js 生态成熟，配合 Clerk 认证和 Cloudinary 存储，能够低成本快速搭建高质量全栈应用。
2. 操作可行性：系统界面设计遵循移动端优先原则，操作逻辑符合主流社交软件习惯，准妈妈群体易于上手。
　　
3.2 功能需求分析
　　
　　根据孕期交流的实际场景，系统需满足以下核心用例：
1. 用户模块：支持注册登录、编辑个人档案（预产期、所在地等）、查看个人主页。
2. 动态广场模块：用户可发布包含文字、图片的动态（如晒B超单、孕妇餐），支持下拉刷新加载。
3. 互动社交模块：支持对动态进行点赞、评论；用户之间可以互相关注（Follow），查看关注列表。
4. 消息通知模块：当收到点赞、评论或关注时，需有红点提示；支持用户间点对点发送文字即时消息。
5. 搜索模块：支持通过关键词搜索特定用户或特定内容的动态帖子。

第四章 系统设计
　　
4.1 系统架构设计
　　
　　系统采用前后端一体化架构（Monorepo 风格）：
* 表现层（Frontend）：由 React 组件构成，分为 Client Components（处理交互）和 Server Components（负责数据获取）。
* 逻辑层（API）：Next.js API Routes 和 Server Actions，处理业务逻辑、权限校验。
* 数据层（Database）：MySQL 关系型数据库，存储用户信息、动态内容等。

4.3 数据库设计 (Database Design)
　　
　　基于业务需求，设计了以下核心数据模型（参考?schema.prisma）：
1. User（用户表）
o 字段：id, username, avatar, cover, description, city, work, createdAt 等。
o 描述：存储基础身份信息，关联所有其他表。
2. Post（动态表）
o 字段：id, desc (内容), img (图片URL), userId (作者), createdAt。
o 描述：存储用户发布的主体内容。
3. Comment（评论表）
o 字段：id, desc, userId, postId。
o 描述：依附于 Post 的评论内容。
4. Like（点赞表）
o 字段：id, userId, postId, commentId。
o 描述：记录点赞行为，实现多态关联（可赞帖子，未来可赞评论）。
5. Follower（关注表）
o 字段：followerId (粉丝), followingId (被关注者)。
o 描述：实现用户间的多对多社交关系。
6. Comparison / Message（私信相关表）
o 字段：senderId, text, isRead, conversationId。
o 描述：存储聊天记录及已读状态。

第七章 总结与展望
　　
7.1 总结
　　
　　本文依据软件工程的思想，完成了一个基于 Next.js 技术栈的准妈妈孕期交流平台的设计与开发。系统实现了从前端 UI 到后端数据库的全栈开发流程，解决了动态实时交互、多媒体存储、复杂社交关系查询等技术问题。该平台界面清新温馨，功能贴合孕期社交需求，具有较好的实用性。
　　
7.2 不足与展望
　　
　　尽管系统已具备核心功能，但仍存在改进空间：
1. 实时性优化：目前消息通知主要依赖刷新和短轮询，未来计划引入 WebSocket (Socket.io) 实现真正的毫秒级即时通讯。
2. 智能化扩展：计划接入 AI 大模型接口，提供智能“孕期问答助手”，自动回答准妈妈常见的饮食禁忌等问题。
3. 数据分析：增加后台管理大屏，统计每日活跃用户数和热门话题趋势，辅助运营决策。
